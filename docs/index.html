Patch

A lightweight, extensible code editor built on Electron & Monaco.

About Patch

Patch is a developer-focused code editor with a plugin system, integrated terminal, AI chat, git integration, and full Monaco editor power — all running locally.

Monaco Editor

Same engine as VS Code — syntax highlighting, IntelliSense, multi-cursor, find & replace.

Plugin System

Extend with custom sidebar panels, commands, shell integration, and Monaco extensions.

AI Chat

Built-in AI assistant with multi-provider support (OpenAI, Anthropic, OpenRouter, Ollama, and more).

Integrated Terminal

Full xterm terminal with node-pty, supports multiple tabs.

Git Integration

View changed files, stage/unstage, commit, push, pull — all from the sidebar.

File Management

Create, rename, delete, move files and folders. Drag-and-drop support. Hidden file toggling.

Getting Started

Installation

git clone https://github.com/your-username/patch.git
cd patch
npm install
npm start

Build for Distribution

npm run build           # esbuild — outputs to dist/
npm run dist:linux      # AppImage
npm run dist:mac        # DMG
npm run dist:win        # NSIS installer

Open a Folder

Click Open Folder on the welcome screen, or drag a folder onto the window. The file explorer shows your project tree.

Editor

Patch uses the Monaco Editor — the same engine that powers VS Code.

Features

  • Syntax highlighting — JavaScript, TypeScript, Python, JSON, HTML, CSS, Markdown, and 100+ languages
  • IntelliSense — autocomplete, parameter hints, go-to-definition, find references
  • Multi-cursor — Alt+Click to add cursors, Ctrl+Alt+Up/Down for column selection
  • Find & Replace — Ctrl+F for in-file, Ctrl+Shift+F for cross-file search
  • Minimap — toggle in settings or via command palette
  • Breadcrumbs — show file path above the editor
  • Zen Mode — hides sidebar and panels for distraction-free editing

View changed files grouped by status (modified, added, deleted, untracked). Stage/unstage files, commit with message, push/pull. Auto-polls every 5 seconds.

Plugin API — Quick Start

Plugins extend Patch with custom sidebar panels, commands, shell integration, Monaco extensions, and event hooks.

User Script — ~/.patch/init.js

Runs automatically on startup. No manifest needed.

// ~/.patch/init.js
patch.ui.addStatus('Hello from init.js');
patch.commands.register('my-cmd', 'My Command', function() {
  patch.editor.setContent('Hello from plugin!');
});

Shareable Plugin — ~/.patch/plugins/<name>/

Create a folder with manifest.json and index.js:

// manifest.json
{
  "id": "my-plugin",
  "name": "My Plugin",
  "version": "1.0.0",
  "description": "Does something cool"
}

// index.js
function activate(patch) {
  patch.ui.addStatus('Plugin loaded!');
}

Open the Plugins panel (Ctrl+Shift+J) to toggle or reload. Commands appear in the palette. Sidebar views get activity bar buttons.

patch.editor

Control the active editor — read/write content, manage tabs, access Monaco directly.

patch.exec

SignatureReturns
patch.exec(command)Promise<string> — stdout

Runs a shell command in the main process via child_process.exec.

patch.exec('df -h / | tail -1').then(function(out) {
  patch.ui.addStatus('Disk: ' + out.trim());
});

patch.monaco

Raw Monaco global. Register languages, themes, providers, actions, keybindings.

patch.monaco.languages.registerHoverProvider('javascript', {
  provideHover: function(model, position) {
    return { contents: [{ value: '**Plugin hover!**' }] };
  }
});

Plugin Lifecycle

  1. Scan — Reads ~/.patch/init.js and scans ~/.patch/plugins/ for manifest.json
  2. API creation — Fresh patch object with empty command/view registries
  3. Execution — init.js runs raw. Plugin index.js: export stripped, activate(patch) called
  4. Reload — Clears all state, re-scans, re-executes. Old API references become stale.
Tip: Persist data with patch.fs.writeFile(). Closure variables reset on reload.

Examples

GPU Monitor

function activate(patch) {
  var data = { gpu: '', mem: '', cpu: '' };

  function fetch() {
    patch.exec('lspci 2>/dev/null | grep -i "vga\\|3d\\|display" | ' +
      'head -1 | sed \'s/.*: //;s/ (rev.*//\' || echo unknown')
      .then(function(gpu) { data.gpu = gpu; return patch.exec('free -m | awk \'/Mem:/ {print $3"/"$2" MB"}\''); })
      .then(function(mem) { data.mem = mem; return patch.exec('top -bn1 | head -3'); })
      .then(function(cpu) { data.cpu = cpu; });
  }

  function render() {
    var cpuLine = data.cpu.split('\n').find(function(l) { return l.indexOf('%Cpu') >= 0; });
    return '<h3>GPU Monitor</h3><table>'
      + '<tr><th>GPU</th><td>' + data.gpu.trim() + '</td></tr>'
      + '<tr><th>Memory</th><td>' + data.mem.trim() + '</td></tr>'
      + '<tr><th>CPU</th><td>' + (cpuLine ? cpuLine.trim() : 'N/A') + '</td></tr></table>';
  }

  patch.ui.registerSidebarView('gpu', 'GPU Monitor',
    '<svg viewBox="0 0 24 24" width="18" height="18" fill="none" stroke="currentColor" stroke-width="2">' +
    '<rect x="2" y="7" width="20" height="10" rx="2"/><circle cx="12" cy="12" r="2"/>' +
    '<path d="M16 12h4M4 12h4"/></svg>', render);

  patch.commands.register('show-gpu', 'Show GPU Usage', function() {
    fetch();
    setInterval(fetch, 3000);
    patch.ui.addStatus('GPU monitor started');
  });
}

JSON Formatter

function activate(patch) {
  patch.commands.register('format-json', 'Format JSON', function() {
    try {
      var formatted = JSON.stringify(JSON.parse(patch.editor.getContent()), null, 2);
      patch.editor.setContent(formatted);
      patch.ui.addStatus('Formatted');
    } catch (e) { patch.ui.addStatus('Invalid JSON'); }
  });
}

Tips & Ideas

Keep render() fast

Called on every sidebar re-render. Do I/O in intervals/commands, store results in closure vars.

Graceful shell

Use 2>/dev/null || echo fallback to avoid unhandled rejections.

No require()

Plugins run in a sandbox. Use patch.exec() for CLI tools instead.

Unique IDs

Command and view IDs are deduplicated. Second registration is silently ignored.

SVG icons

18×18 viewBox. Use currentColor for theme-aware icons.

Persist state

State resets on reload. Use patch.fs.writeFile() to save data to disk.

Plugin Ideas

  • Syntax checker — run eslint/ruff via patch.exec()
  • Color picker — sidebar view with HTML color input
  • Live Markdown preview — render markdown in a sidebar
  • Todo highlighter — scan for TODO/FIXME comments
  • Git blame — annotations via git blame
  • Bookmarks — save/restore cursor positions
  • AI prompt templates — inject custom prompts into the chat
  • Project notes — save markdown notes per folder
  • Keybinding layer — custom shortcuts via Monaco API
  • Bulk renamer — regex rename files in the project
