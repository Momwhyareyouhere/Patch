Patch

A lightweight, extensible code editor built on Electron & Monaco.

About Patch

Patch is a developer-focused code editor with a plugin system, integrated terminal, AI chat, git integration, and full Monaco editor power — all running locally.

Quick Start — 5-Minute Tutorial

Get something done in Patch right now.

Open a Folder

Press Ctrl+O (or Cmd+O on Mac) to open a project folder. You can also drag a folder onto the window. The file explorer shows your project tree in the sidebar.

Edit a File

Click any file in the explorer to open it. Start typing — Monaco gives you syntax highlighting, autocomplete, and IntelliSense. Press Ctrl+S to save.

Search Across Files

Press Ctrl+Shift+F to open the search sidebar. Type a query to find matching lines. Use .* for regex, Aa for case-sensitive matching.

Run a Git Commit

Open Source Control in the sidebar. Type a commit message and click Commit. Push to sync with remote.

Open the Command Palette

Press Ctrl+Shift+P. Type any command — "Toggle Terminal", "Format Document", "Toggle Minimap". Plugin commands appear here too.

Diff Editor

Compare any two files side-by-side using Monaco's built-in diff editor.

How to Use

  1. Right-click a file in the explorer and select Select for Compare

Multi-root Workspaces

Work with multiple project folders in a single window.

Adding Folders

  • Open Settings (Ctrl+,) and use the Workspace Folders section
  • Run Add Folder to Workspace from the command palette
  • Each folder appears as a separate root in the file explorer, with the active folder highlighted

Removing Folders

Click the X button next to any folder in the workspace list within settings. The status bar shows the current folder count (e.g. project-name +2).

Workspace folders are saved to ~/.patch-settings.json and survive restarts.

Integrated Terminal

Full terminal experience powered by xterm.js and node-pty. Toggle with Ctrl+`.

Features

  • Multiple tabs — each tab is an independent shell session
  • Create / close tabs — plus/minus buttons in the terminal header, or Ctrl+Shift+` for a new tab
  • Copy & paste via terminal context menu or standard clipboard shortcuts
  • 256-color & true color support for modern CLI tools
  • Mouse events for programs like tmux, htop, nano
  • Auto-scroll lock when you scroll up; auto-scroll resumes when at the bottom
  • Resizable panel — drag the divider up/down
  • Close on exit — tab closes automatically when the shell exits
  • Persistent session — terminal survives file navigation and editor use

Git Integration

Extension Marketplace

Browse, install, and manage plugins from built-in and custom repositories.

Built-in Registry

Patch ships with a default marketplace registry at https://patch-marketplace.onrender.com/api/v1/plugins. Plugins are listed with name, description, author, version, download count, and a screenshot.

Actions

  • Install — click the Install button on any plugin card
  • Uninstall — remove a plugin from the Installed tab
  • Toggle — enable/disable installed plugins without removing them
  • Reload All — re-scan the ~/.patch/plugins directory for local plugins
  • Filter — search by plugin name across all available plugins

Plugin Metadata

Each plugin includes a name, version, description, author, and main entry point. These are displayed in the Installed tab for every installed plugin.

Custom Registries

You can add custom marketplace URLs in the settings. The marketplace will fetch from all configured registries and merge the results.

Plugin Development

A Patch plugin is a folder with a main.js file that exports an activate function.

Quick Start

// ~/.patch/plugins/my-plugin/main.js
module.exports = {
  activate: function(patch) {
    console.log('Hello from my plugin!');
  }
};

Folder Structure

~/.patch/plugins/my-plugin/
  main.js
  package.json   # optional, provides metadata

Lifecycle

  1. Place your plugin folder in ~/.patch/plugins/
  2. Click Reload All in the Plugins panel, or restart Patch
  3. If activate throws, the plugin is marked as errored and skipped

package.json (optional)

{
  "name": "my-plugin",
  "version": "1.0.0",
  "description": "Does something cool",
  "author": "You",
  "main": "main.js"
}

patch.commands

Register new commands and execute existing ones. Prefix with your plugin name to avoid collisions.

MethodDescription
register(id, handler)Register a command. handler receives args from the command palette or other callers
execute(id, args?)Execute a command by ID (built-in or registered by any plugin)
remove(id)Unregister a previously registered command
patch.commands.register('my-plugin.hello', function() {
  patch.ui.showNotification('Hello from my-plugin!');
});

patch.exec

Run shell commands in the project root directory.

MethodReturnsDescription
exec(command)Promise<{ stdout, stderr, code }>Run a shell command. Working directory is the project root.
spawn(command, args)ChildProcessSpawn a long-running process with stdio pipes.
patch.exec.exec('npm test').then(function(result) {
  patch.ui.showNotification(result.code === 0 ? 'Tests passed!' : 'Tests failed',
    result.code === 0 ? 'success' : 'error');
});
exec captures output into memory. For streaming or long-running processes, use spawn instead.

Plugin Lifecycle

A plugin goes through four stages: discovered, loaded, activated, and optionally deactivated.

Discovery

On startup (or "Reload All"), Patch scans ~/.patch/plugins/*/ for plugin folders. Each folder must contain a main.js (or the file specified in package.json). Invalid folders are skipped with an error logged to the console.

Loading

Each plugin's main.js is require()'d. If the module throws during load, the plugin is marked as errored and is not activated. The error message is shown in the Plugins panel.

Activation

If the module exports an activate function, it is called with the patch API object. The function may return a deactivation function:

module.exports = {
  activate: function(patch) {
    var timer = setInterval(function() {
      console.log('tick');
    }, 1000);
    return function() {
      clearInterval(timer);
    };
  }
};

Deactivation

When a plugin is toggled off or Patch shuts down, the optional deactivation function returned by activate is called. Commands registered via patch.commands are automatically unregistered. Sidebar views are automatically removed.

Error State

If activate throws, the plugin is marked errored. Errors are displayed in the Plugins panel. Fix the issue in the plugin folder and click "Reload All" to re-activate.

Examples

Ready-to-use plugin examples to learn from.

Status Bar Word Counter

// ~/.patch/plugins/word-count/main.js
module.exports = {
  activate: function(patch) {
    function update() {
      var text = patch.editor.getValue();
      var words = text.trim() ? text.trim().split(/\s+/).length : 0;
      patch.ui.showNotification('Words: ' + words);
    }
    patch.commands.register('word-count.show', update);
  }
};

Auto-backup on Save

module.exports = {
  activate: function(patch) {
    patch.on('editor.save', function() {
      var content = patch.editor.getValue();
      var path = patch.editor.getFilePath();
      var backupPath = path + '.backup';
      patch.fs.writeFile(backupPath, content);
    });
  }
};

Git Status Checker

module.exports = {
  activate: function(patch) {
    setInterval(function() {
      patch.git.status().then(function(s) {
        if (s.files.length > 0) {
          patch.ui.showNotification(s.files.length + ' changed files', 'warning');
        }
      });
    }, 30000);
  }
};

Minimal Sidebar Panel

module.exports = {
  activate: function(patch) {
    patch.ui.registerSidebarView('todo-list', 'Todo', 'check', function(el) {
      el.innerHTML = '<ul><li>Write plugin</li><li>Profit</li></ul>';
    });
  }
};

Tips & Ideas

Ideas for plugins you can build with the Patch API.

  • Project Dashboard — Use patch.fs and patch.ui.registerSidebarView to show build status, test results, or TODO counts.
  • Snippet Expander — Listen to editor.change, detect shortcuts like :log and expand them to console.log().
  • Format-on-save — Hook into editor.save, run patch.exec.exec('prettier --write ' + path).
  • Color Preview — Use patch.editor.setDecorations to highlight color hex values with their actual color as a background.
  • Task Runner — Register commands for npm run build, npm test, etc. Show output via notifications.
  • Inline Translator — Select text, execute a command that calls an external API and replaces the selection with translated text.
  • Git Blame Annotations — Use patch.git.log() and decorations to show the last commit info for each line in the gutter.
  • Custom Theme Switcher — Register a command that cycles through multiple accent colors or editor themes.
  • Bookmark Manager — Save line positions with labels and jump between them via the command palette.
  • Live Markdown Preview — Register a sidebar view that renders the current markdown file as HTML in real-time.
  • Pair Programming — Use patch.internal IPC to sync editor content with a remote peer.
