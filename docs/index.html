Quick Start

Plugins let you extend Patch with custom sidebar panels, editor commands, shell integration, and more. Every plugin receives a patch API object.

User Script — ~/.patch/init.js

Runs automatically on startup. No manifest needed.

// ~/.patch/init.js
patch.ui.addStatus('Hello from init.js');
patch.commands.register('my-cmd', 'My Command', function() {
  patch.editor.setContent('Hello from plugin!');
});

Shareable Plugin — ~/.patch/plugins/<name>/

Create a folder with two files:

~/.patch/plugins/my-plugin/ ├── manifest.json └── index.js

manifest.json

{
  "id": "my-plugin",
  "name": "My Plugin",
  "version": "1.0.0",
  "description": "Does something cool"
}

index.js

function activate(patch) {
  // Your plugin code here
  patch.ui.addStatus('Plugin loaded!');
}

Open the Plugins panel (Ctrl+Shift+J) to toggle plugins on/off or hit Reload to re-scan. Commands appear in the command palette (Ctrl+Shift+P). Sidebar views appear as activity bar buttons.

manifest.json

FieldTypeDescription
idstringUnique identifier (kebab-case)
namestringDisplay name
versionstringSemver version
descriptionstring (optional)Short description shown in the plugin panel

API Reference

patch.editor

Control the active editor tab — read/write content, manage tabs, access Monaco directly.

MethodReturnsDescription
getActiveTab()OpenTab | nullCurrently visible tab object
getContent()stringFull text of the active editor
setContent(text)voidReplace editor content entirely
getSelection()Selection | nullMonaco selection range
getMonacoEditor()editor | nullRaw Monaco editor instance for advanced use
createOutputTab(name, content, language?)voidOpen a virtual read-only tab (output:// path, save is blocked)

OpenTab shape

{
  path: string;          // Unique path (real fs path or "output://..." for virtual tabs)
  name: string;          // Display name in the tab bar
  language: string;      // Monaco language identifier (e.g. "javascript", "json")
  content: string;       // Current editor content
  savedContent: string;  // Content at last save (used for dirty detection)
  isDirty: boolean;      // Whether content differs from savedContent
}

getActiveTab()

Get the currently focused tab. Returns null if no tabs are open.

getContent()

Read the full text from the active Monaco editor model.

setContent(text)

Replace the entire editor content. Marks the tab as dirty.

createOutputTab(name, content, language?)

Creates a virtual tab (path output://<name>). If the tab exists, content is replaced. Saves are silently ignored.

// Insert text at cursor position
var ed = patch.editor.getMonacoEditor();
if (ed) {
  var sel = ed.getSelection();
  ed.executeEdits('plugin', [{ range: sel, text: 'Hello!' }]);
  ed.focus();
}

// Open a build output tab
patch.editor.createOutputTab('Build Output',
  'Compilation successful\n0 errors\n2 warnings',
  'plaintext'
);
Tip: Use getMonacoEditor() to access the full Monaco API — add keybindings, register providers, manipulate selections, or run any editor action.

patch.ui

Surface your plugin in the Patch UI — status bar messages and custom sidebar panels.

MethodDescription
addStatus(text)Show a message in the status bar (auto-clears after 3 seconds)
registerSidebarView(id, label, icon, render)Register a custom sidebar panel with its own activity bar button

registerSidebarView(id, label, icon, render)

ParamTypeDescription
idstringUnique view identifier. Used as the sidebar state key.
labelstringTooltip text and sidebar header label.
iconstringRaw SVG markup for the activity bar button (18×18 viewBox recommended).
render() => stringCalled when sidebar is visible. Return HTML string to render.
HTML in sidebar: The return value of render() is rendered via dangerouslySetInnerHTML. Use var(--text-primary), var(--text-muted), var(--border-color), var(--accent), and var(--sidebar-bg) CSS variables for theme-compliant styling. Tables, headings, code, links, and paragraphs all work. 
// Status bar
patch.ui.addStatus('Plugin initialized');

// Sidebar panel with live data
var data = { temperature: '--' };

patch.ui.registerSidebarView('sensor-view', 'Sensors',
  '<svg viewBox="0 0 16 16" width="18" height="18" fill="currentColor">' +
  '<circle cx="8" cy="8" r="6"/><circle cx="8" cy="8" r="2" fill="var(--accent)"/>' +
  '</svg>',
  function() {
    return '<h3>Temperature</h3>'
      + '<p>CPU: <b>' + data.temperature + '&deg;C</b></p>';
  }
);

patch.commands.register('start-sensors', 'Start Sensors', function() {
  setInterval(function() {
    patch.exec('sensors -j 2>/dev/null | jq -r ".. | .temp1_input? // empty" || echo --')
      .then(function(v) { data.temperature = v.trim(); });
  }, 2000);
});

patch.commands

Register commands that appear in the command palette (Ctrl+Shift+P).

MethodDescription
register(id, label, action, shortcut?)Register a command. id must be unique — duplicate ids are silently ignored.
ParamTypeDescription
idstringUnique command ID (kebab-case)
labelstringDisplay name in the palette
action() => voidCalled when the user selects the command
shortcutstring (optional)Keyboard shortcut hint (display only — does not bind the key)
patch.commands.register('hello-world', 'Say Hello', function() {
  patch.ui.addStatus('Hello, world!');
});

patch.exec

Run shell commands. Executes in the main process via child_process.exec.

SignatureReturns
patch.exec(command: string)Promise<string> — stdout of the command
// Get disk usage
patch.exec('df -h / | tail -1').then(function(out) {
  patch.ui.addStatus('Disk: ' + out.trim());
});

// Handle errors gracefully
patch.exec('which nvidia-smi 2>/dev/null || echo "not found"')
  .then(function(out) {
    if (out.trim() === 'not found') {
      patch.ui.addStatus('No NVIDIA GPU detected');
    }
  });
Tip: Append 2>/dev/null || echo "fallback" to avoid unhandled promise rejections from commands that fail.

patch.fs

Read and write files on disk, get directory trees.

MethodReturnsDescription
readFile(path)Promise<string>Read file contents as a string
writeFile(path, content)Promise<boolean>Write content to a file (creates/overwrites)
getFileTree(dirPath, showHidden?)Promise<FileNode[]>Recursive directory listing

FileNode shape

{
  path: string;
  name: string;
  type: 'file' | 'directory';
  children?: FileNode[];  // Only for directories
}
// Save plugin data
patch.fs.writeFile('/home/user/.patch/data.json',
  JSON.stringify({ lastRun: Date.now() })
);

// Read it back
patch.fs.readFile('/home/user/.patch/data.json').then(function(data) {
  var config = JSON.parse(data);
  console.log(config.lastRun);
});

patch.git

Query git repository status. Operates on the currently opened folder.

MethodReturnsDescription
status(dir)Promise<GitFile[]>List of changed files with their status
branch(dir)Promise<string | null>Current branch name (or null if not a git repo)

GitFile shape

{ path: string; status: string }
// status values: M (modified), A (added), D (deleted), R (renamed), ?? (untracked)
var root = patch.internal.rootPath;
if (root) {
  patch.git.status(root).then(function(files) {
    patch.ui.addStatus(files.length + ' changed files');
  });
  patch.git.branch(root).then(function(branch) {
    if (branch) patch.ui.addStatus('On branch: ' + branch);
  });
}

patch.monaco

Access the raw Monaco editor global for advanced editor customization — register languages, themes, providers, actions, and keybindings.

// Register a hover provider for JavaScript
var monaco = patch.monaco;
if (monaco) {
  monaco.languages.registerHoverProvider('javascript', {
    provideHover: function(model, position) {
      return {
        contents: [{ value: '**Hello from plugin!**' }]
      };
    }
  });
}

// Register a custom action
monaco.editor.registerEditorAction({
  id: 'plugin-uppercase',
  label: 'Uppercase Selection',
  keybindings: [monaco.KeyMod.CtrlCmd | monaco.KeyCode.KeyU],
  run: function(ed) {
    var sel = ed.getSelection();
    var text = ed.getModel().getValueInRange(sel);
    ed.executeEdits('plugin', [{ range: sel, text: text.toUpperCase() }]);
  }
});
Note: patch.monaco is a getter that returns window.monaco. It's available immediately since Monaco loads before plugins execute.

patch.on / patch.off

Subscribe to editor lifecycle events.

MethodDescription
patch.on(event, callback)Subscribe to an event
patch.off(event, callback)Unsubscribe from an event
EventArgumentsDescription
'file:open'(path, fileInfo)Fired when a file is opened in the editor. fileInfo: { path, name, language, content }
'file:save'(path, content)Fired when a file is saved to disk
// Track file history
var history = [];

patch.on('file:open', function(path, info) {
  history.push({ path: path, time: Date.now() });
  if (history.length > 50) history.shift();
});

patch.on('file:save', function(path, content) {
  patch.ui.addStatus('Saved: ' + path.split('/').pop());
});

patch.internal

Read-only access to Patch's internal state. Useful for getting the current folder, file tree, or tabs list.

PropertyTypeDescription
rootPathstring | nullCurrently opened folder path
fileTreeFileNode[]Full file tree of the opened folder
tabsOpenTab[]All currently open tabs
activeTabPathstring | nullPath of the active (focused) tab
editorRefReact.RefObjectThe Monaco editor React ref
// Get the current folder name
var folder = patch.internal.rootPath;
if (folder) {
  var name = folder.split('/').pop();
  patch.ui.addStatus('Working in: ' + name);
}

// Count open tabs
var count = patch.internal.tabs.length;
patch.ui.addStatus(count + ' tabs open');

// Check if a file is currently open
var active = patch.internal.activeTabPath;
var isOpen = patch.internal.tabs.some(function(t) { return t.path === active; });

Plugin Lifecycle

  1. Scan — Patch reads ~/.patch/init.js and scans ~/.patch/plugins/ for folders containing manifest.json
  2. API creation — A fresh patch API object is created with empty command and view registries
  3. Execution
    • init.js — Raw code executed via new Function('patch', code)
    • Plugin index.js — export keywords are stripped, then activate(patch) is called
  4. Reload — From the Plugins panel: all plugin state is cleared (commands, sidebar views), the filesystem is re-scanned, and all enabled plugins re-execute with a fresh API object
Important: Plugin state (closure variables, intervals, timeouts) is lost on reload. The API object is replaced — old references to patch become stale. Use patch.fs to persist data to disk.

Examples

GPU Monitor

Full sidebar panel showing GPU model, memory usage, and CPU stats. Updates every 3 seconds.

Location: ~/.patch/plugins/gpu-monitor/

function activate(patch) {
  var interval = null;
  var data = { gpu: '', mem: '', cpu: '' };

  function fetch() {
    patch.exec('lspci 2>/dev/null | grep -i "vga\\|3d\\|display" | ' +
      'head -1 | sed \'s/.*: //;s/ (rev.*//\' || echo "unknown"')
      .then(function(gpu) {
        data.gpu = gpu;
        return patch.exec('free -m 2>/dev/null | awk \'/Mem:/ {print $3"/"$2" MB"}\'');
      })
      .then(function(mem) {
        data.mem = mem;
        return patch.exec('top -bn1 2>/dev/null | head -3');
      })
      .then(function(cpu) { data.cpu = cpu; });
  }

  function render() {
    var cpuLine = data.cpu.split('\n').find(function(l) {
      return l.indexOf('%Cpu') >= 0;
    });
    return '<h3>GPU Monitor</h3>'
      + '<table>'
      + '<tr><th>GPU</th><td>' + data.gpu.trim() + '</td></tr>'
      + '<tr><th>Memory</th><td>' + data.mem.trim() + '</td></tr>'
      + '<tr><th>CPU</th><td>' + (cpuLine ? cpuLine.trim() : 'N/A') + '</td></tr>'
      + '</table>'
      + '<p style="color:var(--text-muted);font-size:11px">'
      + 'Updated ' + new Date().toLocaleTimeString() + '</p>';
  }

  patch.ui.registerSidebarView('gpu-monitor-view', 'GPU Monitor',
    '<svg viewBox="0 0 24 24" width="18" height="18" ' +
    'fill="none" stroke="currentColor" stroke-width="2">' +
    '<rect x="2" y="7" width="20" height="10" rx="2"/>' +
    '<circle cx="12" cy="12" r="2"/>' +
    '<path d="M16 12h4M4 12h4"/></svg>',
    render
  );

  patch.commands.register('show-gpu', 'Show GPU Usage', function() {
    if (interval) {
      clearInterval(interval);
      interval = null;
      fetch();
      patch.ui.addStatus('GPU monitor stopped');
      return;
    }
    fetch();
    interval = setInterval(fetch, 3000);
    patch.ui.addStatus('GPU monitor started (every 3s)');
  });
}

Hello World

A minimal plugin to demonstrate the boilerplate.

// ~/.patch/plugins/hello-world/index.js
function activate(patch) {
  patch.commands.register('hello-world', 'Hello World', function() {
    patch.ui.addStatus('Hello, world!');
  });

  patch.on('file:open', function(path, info) {

Snippet Injector

A sidebar panel that shows code snippets. Clicking one inserts it at the cursor.

function activate(patch) {
  var snippets = [
    { name: 'Console log', code: 'console.log($1);' },
    { name: 'Function', code: 'function $1($2) {\n  $3\n}' },
    { name: 'Arrow fn', code: 'const $1 = ($2) => $3;' },
    { name: 'Import', code: 'import { $1 } from "$2";' },
  ];

  function render() {
    var html = '<h3>Snippets</h3><ul style="list-style:none;padding:0">';
    snippets.forEach(function(s) {
      html += '<li onclick="patch_insert(\'' + s.code.replace(/'/g, "\\'")
        + '\')" style="cursor:pointer;padding:4px 8px;margin:2px 0;' +
        'border-radius:4px;background:var(--bg3)">'
        + s.name + '</li>';
    });
    return html + '</ul>';
  }

  // We expose a global function since the sidebar HTML runs in a different scope
  window.patch_insert = function(code) {
    var ed = patch.editor.getMonacoEditor();
    if (ed) {
      var sel = ed.getSelection();
      ed.executeEdits('snippet', [{ range: sel, text: code }]);
      ed.focus();
    }
  };

  patch.ui.registerSidebarView('snippets', 'Snippets',
    '<svg viewBox="0 0 16 16" width="18" height="18" fill="currentColor">' +
    '<path d="M4.5 2.5a.5.5 0 01.5.5v3a.5.5 0 01-.5.5H2a.5.5 0 01-.5-.5V3a.5.5 0 01.5-.5h2.5zM2 7.5a.5.5 0 01.5.5v3a.5.5 0 01-.5.5H.5a.5.5 0 010-1H1V8a.5.5 0 01.5-.5H2zm8-5a.5.5 0 01.5-.5H13a.5.5 0 01.5.5v3a.5.5 0 01-.5.5h-2.5a.5.5 0 01-.5-.5V3a.5.5 0 01.5-.5h.5V2.5zM7 3.5a.5.5 0 01.5-.5h1a.5.5 0 010 1h-1a.5.5 0 01-.5-.5zM7 8a.5.5 0 01.5-.5h1a.5.5 0 010 1h-1A.5.5 0 017 8zm-3.5.5a.5.5 0 000 1h1a.5.5 0 000-1h-1z"/>' +
    '</svg>',
    render
  );
}

JSON Formatter

A command-only plugin that formats JSON in the active editor.

function activate(patch) {
  patch.commands.register('format-json', 'Format JSON', function() {
    var text = patch.editor.getContent();
    try {
      var formatted = JSON.stringify(JSON.parse(text), null, 2);
      patch.editor.setContent(formatted);
      patch.ui.addStatus('JSON formatted');
    } catch (e) {
      patch.ui.addStatus('Invalid JSON: ' + e.message);
    }
  });
}

Tips & Patterns

Keep render() fast

render() is called every time the sidebar re-renders. Do heavy work in intervals or commands, store results in closure variables, and keep render() a pure read.

Use var

const/let work in the sandbox, but var is safer for the new Function() context and consistent with existing plugins.

Graceful shell commands

Use 2>/dev/null || echo 'fallback' to avoid unhandled rejections. Chain .catch() for robustness.

No export in index.js

Use function activate(patch) { ... }. export is stripped automatically, but it's cleaner without it.

Icon design

Use 18×18 viewBox for activity bar buttons. Use currentColor for stroke/fill so icons respect the theme.

Persist state

Plugin state resets on reload. Use patch.fs.writeFile() to persist data to ~/.patch/ and read it back in activate().

Unique IDs

Command IDs and sidebar view IDs are deduplicated. If a plugin registers the same ID twice, the second registration is silently ignored.

No require()

Plugins run in a sandbox without Node.js require(). Use patch.exec() to invoke CLI tools instead.

Plugin Ideas

  • Syntax checker — run eslint, ruff, or gofmt via patch.exec() and display problems
  • Clipboard manager — track file:save events and maintain a clipboard history sidebar
  • Color picker — a sidebar view with an HTML color input that inserts hex values
  • Live Markdown preview — render markdown in a sidebar view using a lightweight parser
  • Project notes — save/load markdown notes scoped to the current folder
  • Time tracker — log which files you edit and for how long, export to CSV
  • AI prompt templates — inject custom system prompts or user messages into the AI chat
  • Keybinding layer — use patch.monaco.editor.addEditorAction() to add custom keyboard shortcuts
  • Todo highlighter — scan open files for TODO and FIXME comments, show in sidebar
  • Bookmarks — save/restore cursor positions per file, navigate with a sidebar list
  • Git blame viewer — run git blame on the active file and show annotations
  • Bulk renamer — sidebar panel with regex rename for files in the project
